## Interaction preferences

When working with files in this repository:

- Cite the source file when referencing context (e.g., "from `context/01_identity/brand.md`")
- Ask before creating new files
- Prefer updating existing files over creating duplicates
- Commit with meaningful messages that explain why, not just what
- Flag anything that looks like a secret before writing it to a file

---

## Secrets policy

No secrets in context files. API keys, tokens, passwords, and PII belong in a password manager or secrets manager — not here. The git hooks in `.githooks/` enforce this at commit and push time.
